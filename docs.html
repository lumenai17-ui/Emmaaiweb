1. Inicio Rapido

Acceso Anticipado (Usuarios)

La forma mas rapida de empezar.

  • Descarga el APK desde la pagina de compra
  • Instala habilitando "Origenes desconocidos" en Ajustes
  • Abre E.M.M.A. y acepta los permisos
  • Ve a Settings y configura tu provider de IA
  • Abre La Forja para crear tu primer agente

Tiempo estimado: 2 minutos

GitHub (Desarrolladores)

Para quienes quieren compilar, auditar o contribuir.

# Clonar el repositorio
git clone https://github.com/lumenai17-ui/emma-ai.git

# Abrir en Android Studio
# Requiere JDK 17 + Android SDK 35

# Compilar debug
./gradlew assembleDebug

# Compilar release (requiere keystore)
./gradlew assembleRelease

La version GitHub no incluye APK pre-compilado. Debes compilar desde el codigo fuente.

Configurar tu Primer Provider de IA

E.M.M.A. necesita al menos un cerebro para funcionar. Elige uno:

Cloud (OpenRouter)

  • Crea cuenta en openrouter.ai
  • Genera una API key
  • En E.M.M.A. → Settings → LLM Provider → OpenRouter
  • Pega tu API key
  • Selecciona modelo (GPT-4o, Claude, Gemini, etc.)

Local (Ollama)

  • Instala Ollama en tu PC/servidor
  • Descarga un modelo: ollama pull gemma3
  • En E.M.M.A. → Settings → LLM Provider → Ollama
  • Ingresa la IP de tu servidor (ej: 192.168.1.100:11434)

2. Arquitectura Multi-Agente

E.M.M.A. es un orquestador de enjambres de agentes IA (Agent Swarm Hub) con protocolo A2A (Agent-to-Agent).

ModuloDescripcion
EmmaEngine (Supervisor)Directora del sistema. Evalua y delega tareas sin resolver directamente, reduciendo alucinaciones
Agent Swarm HubInterfaz tipo WhatsApp para multiples agentes con threads independientes
Koog Engine (A2A)Protocolo de enrutamiento que permite handoff de memoria comprimida entre agentes
Agent Factory ("La Forja")Creador dinamico de agentes con System Prompt, icono y seleccion de cerebro
Compressive HandoffTransferencia de contexto entre agentes bajo un mismo thread
Pre-Seed AutomaticoInyecta agentes pilares ("E.M.M.A. Chat", "Live Vision", "Deep Voice") al primer inicio
Hermes TunnelWebSocket persistente (OkHttp) para comunicacion bidireccional con servidor maestro

Paquetes de Codigo Fuente (18 modulos)

  • a2a/ — Protocolo Agent-to-Agent
  • core/ — Logica central y configuracion
  • database/ — Room DB (AgentConfig, ChatThread, Groups)
  • email/ — Servicio IMAP/SMTP directo
  • files/ — AttachmentManager + PublicFileWriter
  • google/ — Auth + Calendar + Drive + Gmail + Tasks
  • llm/ — Providers (OpenRouter, Ollama, Gemma, Custom)
  • memory/ — Gestion de memoria por agente
  • network/ — Conectividad y fetchers
  • plugins/ — Sistema de plugins (21 builtins)
  • security/ — EncryptedSharedPreferences, SecurePrefs
  • service/ — Background services
  • telegram/ — TelegramBotService (long-polling)
  • telemetry/ — Scanner ambiental + WorkManager
  • tunnel/ — Hermes WebSocket (Client + Manager + Service)
  • ui/ — Screens + Components + Theme
  • vision/ — CameraX + Face Detection + GPS Nav
  • voice/ — Deepgram STT/TTS + ElevenLabs

3. Modelos de IA

Cloud (via OpenRouter)

GPT-4o Claude 3.5+ Gemini 2.5 Pro Llama 3.3 +cientos mas

Acceso via API key de OpenRouter. Compatible con cualquier modelo del catalogo.

Local / Edge (via Ollama)

Gemma 4 Qwen 3 MoE DeepSeek R1 Command R+

Requiere servidor Ollama en red local o remoto. Modelos open-source.

On-Device (100% Offline)

Gemma 4 E2B Gemma 4 E4B

Formato .litertlm. Motor LiteRT-LM 0.9. Inferencia directa en ARM sin internet.

Custom API

Cualquier endpoint compatible con /v1/chat/completions (formato OpenAI). Configura URL base + API key en Settings.

7. Voz e Idiomas

Motores de Voz

  • Deepgram Nova-2 — STT ultra baja latencia
  • Deepgram Aura — TTS premium (ingles)
  • ElevenLabs — Voice cloning multilingue
  • Android TTS nativo — Fallback gratuito del sistema

Cadena de fallback: ElevenLabs → Deepgram → Android TTS

Idiomas

  • Espanol — UI + Voz + Chat (idioma primario)
  • Ingles — UI + Voz + Chat
  • Otros — Adaptivo via Locale.getDefault()

Dual XML (i18n): strings.xml en espanol e ingles con traduccion adaptiva.
